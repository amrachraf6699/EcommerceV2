# Phase 7: Customer Auth and Account

## Goal
Expose the existing customer model and auth foundation through a usable customer-facing experience.

## Scope
- Build customer login and registration.
- Include password recovery if it is part of the release.
- Build account pages:
  - profile
  - addresses
  - orders list
  - order details
- Reuse the storefront design system instead of admin patterns.

## Deliverables
- Auth screens consistent with the storefront identity.
- Account area that works in Arabic RTL and feels customer-facing.
- Address and order history UI powered by real customer data.

## Test Checklist
- Registration, login, and logout work end-to-end.
- Password recovery works if included.
- Customer addresses can be viewed and managed.
- Orders history and order detail pages display correct data.

## Acceptance
- Customer account flow is functional and visually consistent.
- Auth and account pages feel like part of the storefront, not admin reuse.
- Account section is ready for production refinement.
