# Phase 2: Header, Footer, Navigation

## Goal
Build the global storefront chrome so every later page inherits a complete and stable structure.

## Scope
- Build storefront header and footer.
- Add desktop navigation.
- Add mobile drawer navigation.
- Add customer-facing interaction shell:
  - mobile menu open/close
  - hover states
  - active navigation states
- Apply website-wide RTL spacing and alignment rules.

## Deliverables
- Reusable header and footer partials.
- Responsive navigation behavior for mobile and desktop.
- Navigation visuals aligned with the black-and-white editorial identity.

## Test Checklist
- Desktop navigation renders and stays stable across widths.
- Mobile drawer opens, closes, and traps focus correctly if implemented.
- Active state appears correctly for current route.
- Header and footer remain visually consistent across sample pages.

## Acceptance
- Navigation feels intentional, modern, and not like reused admin UI.
- Mobile menu behavior is clean and reliable.
- Header/footer are ready for homepage and catalog pages.
